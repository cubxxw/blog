# 适配器 · 视频脚本（B站 / YouTube）

**触发**："把 `2-master/<slug>.md` 分发成视频脚本"
**产物**：口播脚本（带时间轴+分镜提示）+ 标题/封面文案 + 简介与标签。

---

## 人群与机制
观众用**前 3 秒**决定走留，靠完播率和互动推流。母版要转成**能听懂的口语线性叙事**——不能像读文章。信息密度保留，但节奏、悬念、口播感是新加的。B站重弹幕梗与人设，YouTube 重清晰结构与章节。

## 生成规则

**开头黄金 3 秒（Hook）**
用母版 `pain_points` / `one_liner` 做一句直接戳中或制造悬念的开场，先抛结论/冲突，别铺垫。

**口播脚本**
- 分段，每段标 `[00:00]` 时间轴 + `（分镜：…）` 画面提示。
- 结构：Hook → 为什么值得看（承诺收益）→ 主体分点（母版 `key_takeaways`，每点一小节，配例子/演示）→ 总结 → CTA（三连/订阅 + 下集预告）。
- 语言：口语、短句、能读出声；避免书面长句和生僻术语，术语要顺带解释。
- 每 30-60 秒安排一个"钩子"维持完播（设问、反转、"接下来最关键"）。

**标题 / 封面**
- 标题：钩子 + 关键词，B站可带情绪/梗，YouTube 更直白清晰。
- 封面文案：6-10 字大字 + 一个表情/对比点。

**简介与标签**
- 简介：2-3 句讲清收益 + 章节时间戳（YouTube chapters）+ 回指博客全文链接。
- 标签：垂直词 + 大词混合。

**可选产物**
- 分镜表（镜头/画面/字幕/时长），如果要做正式剪辑。
- 竖屏短视频版（30-60 秒）：只取一个最强 takeaway，用于引流。

## 注意
- 口播稿要"能读"，写完自己念一遍不卡壳为准。
- 演示类内容确保步骤真实可复现。
