## 一、一句话结论

**你的站点技术 SEO 已经做到接近满分（Lighthouse SEO = 100，Best Practices = 100），瓶颈根本不在"技术层"，而在三件事上：**

1. **域名迁移正在进行中**（`nsddd.top` → `cubxxw.com`），这是当前最高风险，处理得当能保住三个月 85 万+ 曝光的历史权重，处理不当会丢失排名。
2. **排名停在第二页**（平均排名 13.2），大量关键词卡在第 11–20 位，只差临门一脚就能上首页。
3. **点击率被"垃圾曝光"严重稀释**（整站 CTR 仅 0.1%）——87 万曝光里很大一部分来自与博客主题无关的长尾噪声词，真正能转化的是技术类关键词。

所以优化重心应该从"继续堆技术 SEO"转向"保迁移、抢首页、提点击率、做内容集群"。

## 三、你已经做对的（不要重复劳动）

这个站点的 SEO 基建在个人博客里属于**顶尖水平**，以下都已经到位、无需再动：

- **完整的 `seo.html` 模板**：canonical、description、keywords、author、Open Graph 全套（type / site_name / locale + alternate / url / title / description / image 1200×630 / published_time / modified_time / article:tag / article:section）、Twitter `summary_large_image` 大图卡。
- **分栏目专属 OG 图**：`ai-technology` / `growth` / `projects` 各有独立分享图，社交分享观感好。
- **结构化数据齐全**：文章页同时输出 `BlogPosting`、`WebSite`（含站内搜索 SearchAction）、`Person`（含 sameAs 社交矩阵）、`BreadcrumbList` 四类 JSON-LD，已在生产环境验证通过。
- **hreflang 正确**：首页与文章页均输出 `en / zh / x-default`，且 sitemap 内也带 `xhtml:link` 备用语言标注。
- **robots.txt 面向未来**：不仅放行 Googlebot/Bingbot，还显式欢迎 AI 答案引擎（GPTBot、ClaudeBot、PerplexityBot、Google-Extended 等）和国内爬虫（Baiduspider、Bytespider、Sogou、YisouSpider），这是很有前瞻性的 **GEO（生成式引擎优化）** 布局。
- **多重订阅出口**：`sitemap.xml` + `news-sitemap.xml` + `llms.txt` / `llms-full.txt`（给大模型的站点地图）+ JSON Feed + OPML。
- **meta robots** 用了 `max-image-preview:large, max-snippet:-1, max-video-preview:-1`，最大化 SERP 展现。
- **性能基建**：字体异步加载（preload + noscript 回退）、preconnect/dns-prefetch 预热、首页 LCP 图 `fetchpriority=high` 预加载、GA4 已接入。

**结论：技术 SEO 这块，继续保持即可，别再往上叠了——边际收益已经很低。**

## 四、真实浏览器测试结果（PageSpeed Insights，移动端）

在 `pagespeed.web.dev` 对 `https://cubxxw.com/` 实测（2026-07-10）：

### 核心指标（实验室数据）

| 指标 | 移动端实测值 | 目标 | 状态 |
|---|---|---|---|
| First Contentful Paint (FCP) | 1.7 s | < 1.8 s | ✅ 达标 |
| **Largest Contentful Paint (LCP)** | **3.3 s** | < 2.5 s | ⚠️ 需改进 |
| Total Blocking Time (TBT) | 0 ms | < 200 ms | ✅ 优秀 |
| Cumulative Layout Shift (CLS) | 0.072 | < 0.1 | ✅ 达标（偏高） |
| Speed Index | 2.6 s | < 3.4 s | ✅ 达标 |

> **注意：CrUX 真实用户字段数据显示 "No Data"** —— 说明站点真实访问量还没达到 Google 采集"实地体验数据"的门槛。这本身是一个信号：需要提高流量规模，Core Web Vitals 的真实评估才会生效。

### 性能优化机会（Lighthouse 诊断，按收益排序）

- **减少未使用的 JavaScript**：约省 111 KiB（最大项）
- **减少未使用的 CSS**：约省 54 KiB
- **图片元素缺少显式 width/height**：这是 CLS 0.072 偏高的主因
- **改进图片投放**：约省 17 KiB（用更优格式/尺寸）
- **消除渲染阻塞资源**：约省 200 ms
- 压缩 JS（约 4 KiB）、1 个长主线程任务、强制回流（forced reflow）

### 无障碍（86 分）失分项 —— 同时影响 SEO 与 AI 抓取

1. **色彩对比度不足**：前景/背景对比未达 WCAG AA（影响可读性评分）。
2. **标题层级非顺序递减**：存在跳级（如 h2 → h4），Google 与屏幕阅读器都依赖标题结构理解内容。
3. **`<li>` 未包裹在 `<ul>/<ol>/<menu>` 内**：无效 HTML 结构。
4. **触控目标过小/间距不足**：移动端可点区域偏小。
5. **ARIA tooltip 缺少可访问名**：见下条。

### Agentic Browsing（AI 可读性，2/3）失分项

- 失败项：**"Accessibility tree is not well-formed"**
- 具体元素：旅行页热力图的 tooltip
  `section#hp-travel > div.hp-travel-block > div#hp-heatmap-wrap > div.hp-heat-tooltip`
  `<div class="hp-heat-tooltip" role="tooltip">` —— 有 `role="tooltip"` 但没有可访问名（aria-label / 文本）。
- 影响：随着 ChatGPT/Claude/Perplexity 等 AI 代理"替用户浏览网页"成为新流量入口，可访问性树是否规范会直接影响 AI 能否正确解析你的页面。**这是面向未来的加分项，值得修。**

### 生产环境 `<head>` 抽查（已验证，全部正确）

首页与文章页（`/ai-technology/posts/advanced-githook-design/`）实测：canonical 正确、hreflang（en/zh/x-default）齐全、文章页 4 类 JSON-LD 全部存在、每页仅 1 个 h1、正文标题 H2→H3 顺序干净、图片带 alt 与尺寸。

> 提示：本地 `public/` 目录是一次**过期构建**（部分文章缺 canonical/hreflang），但**生产环境是正确的**。别被本地旧构建误导——以线上为准。

## 六、优化建议（按优先级）

### 🔴 P0 — 立刻做（保住迁移权重，1 周内）

1. **完成并盯紧域名迁移**
   - 保持旧域 `nsddd.top` 的 301 跳转**至少 180 天**（Google 建议 1 年），期间**不要关停旧域**。
   - GSC 里**两个资源都保留**，每周对比 `nsddd.top`（下降）与 `cubxxw.com`（上升）的曲线，确认权重在转移。
   - 在新域 `cubxxw.com` 资源里**重新提交 sitemap** 并对 5–10 个核心页做"网址检查 → 请求编入索引"，加速新域收录。
   - 逐一核对旧域 813 个有流量页面，确保**每一个都有对应的 301 到新域同名路径**（尤其 `/projects/markitdown/` 这类流量页，绝不能 404）。

2. **单独提交 news-sitemap.xml**
   - `news-sitemap.xml` 已在 robots 里声明，但 GSC 只提交了 `sitemap.xml`。到 Sitemaps 里把 `https://cubxxw.com/news-sitemap.xml` 也加上。

### 🟠 P1 — 高收益（2–4 周）

3. **抢首页：优化"卡在第 11–20 位"的关键词**
   - 用 GSC 过滤"平均排名 8–20"的查询，这些是**离首页最近**的词。对应页面做：补充内容深度、加内部链接（从高权重页指过去）、在正文中自然覆盖这些查询词及其近义词。
   - 优先对象：`markitdown`（英文，72K 曝光）、`gpt researcher`、`mem0`、`langgraph 架构`、`go/golang directives`。

4. **提点击率：重写标题与 description**
   - 对高曝光低 CTR 页面（尤其 markitdown、mem0），把 `<title>` 和 meta description 改成**更贴合搜索意图、含数字/结果承诺**的写法。参考你自己的标杆 `my-hugo`（CTR 10.4%）。
   - 给"How to / 是什么 / 对比"类文章加 **FAQ 结构化数据（FAQPage schema）**，争取 SERP 富媒体展现，天然提升 CTR。

5. **做内容集群（Topic Cluster），而不是追噪声**
   - 你已经在这些真实词上有排名：**Hugo 建站、LangGraph、GPT-researcher、Go directives、NotebookLM、TDD、mem0、markitdown**。
   - 每个主题建一篇"支柱长文（pillar）"+ 若干子文章，互相内链，形成主题权威。**不要**为 MBTI/脑震荡/叶尔羌这类噪声词生产内容——那是数据幻觉，不是机会。

### 🟡 P2 — 打磨（1–2 个月，冲满分体验）

6. **Accessibility 86 → 95+**（同时利好 SEO 和 AI 抓取）
   - 修标题层级跳级（h2→h4 之间补 h3 或调整）。
   - 修色彩对比度不足的文本。
   - 给旅行页热力图 tooltip（`div.hp-heat-tooltip`）加 `aria-label`，同时解决 Agentic Browsing 的 2/3 失分。
   - 修 `<li>` 未被 `<ul>/<ol>` 包裹的结构、放大移动端触控目标。

7. **Performance 90 → 95+ / LCP 3.3s → <2.5s**
   - 给所有正文图片补 `width`/`height` 显式尺寸（消除 CLS 并帮助 LCP）。
   - 清理未使用的 CSS/JS（合计约 165 KiB），按路由/组件拆分。
   - 复查渲染阻塞资源（约 200ms）。

8. **GEO / AI 引擎优化（前瞻布局）**
   - 你已有 `llms.txt`，继续保持并在重要文章开头写清晰的"TL;DR / 摘要段"，方便 AI 摘录引用。
   - 修好可访问性树后，AI 代理抓取质量会同步提升。
